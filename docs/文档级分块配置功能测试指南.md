# 文档级分块配置功能测试指南

本文档描述如何编译、测试和验证文档级分块配置功能。

## 功能概述

文档级分块配置功能允许用户在上传文档时自定义分块参数，覆盖知识库级别的默认配置。配置优先级：

**文档配置 > 知识库配置 > 系统默认值**

## 三、功能测试流程

### 测试 1：单文件上传（使用知识库默认配置）

**步骤**：
1. 打开知识库详情页
2. 点击"上传文档"按钮
3. 选择单个文件
4. 在弹出的配置对话框中，确保选中"使用知识库默认配置"
5. 点击"确认"按钮

**预期结果**：
- 对话框关闭
- 文件开始上传
- 数据库中该文档的 `chunking_config` 字段为 `NULL`
- 文档使用知识库的默认分块配置进行分块

### 测试 2：单文件上传（自定义配置）

**步骤**：
1. 打开知识库详情页
2. 点击"上传文档"按钮
3. 选择单个文件
4. 在弹出的配置对话框中，选中"自定义配置"
5. 修改分块大小（如 1000）、重叠大小（如 100）
6. 点击"确认"按钮

**预期结果**：
- 对话框关闭
- 文件开始上传
- 数据库中该文档的 `chunking_config` 字段为 JSON 对象：
  ```json
  {
    "chunk_size": 1000,
    "chunk_overlap": 100,
    "separators": ["\n\n", "\n", "."],
    "enable_parent_child": false,
    "parent_chunk_size": 4096,
    "child_chunk_size": 384
  }
  ```
- 文档使用自定义配置进行分块

### 测试 3：多文件上传（批量配置）

**步骤**：
1. 打开知识库详情页
2. 点击"上传文档"按钮
3. 选择多个文件（>5 个）
4. 在弹出的配置对话框中，确认显示的文件列表（显示前 5 个 + "And X more files..."）
5. 选中"自定义配置"，设置分块参数
6. 点击"确认"按钮

**预期结果**：
- 对话框关闭
- 所有文件开始上传
- 每个文档都使用相同的自定义配置
- 数据库中每个文档的 `chunking_config` 字段都正确设置

### 测试 4：全局拖拽上传

**步骤**：
1. 从任意页面拖拽文件到应用窗口
2. 在弹出的配置对话框中选择目标知识库
3. 设置自定义分块配置
4. 点击"确认"按钮

**预期结果**：
- 对话框关闭
- 文件上传到指定知识库
- 使用自定义配置进行分块

### 测试 5：父子分块配置

**步骤**：
1. 打开上传配置对话框
2. 选中"自定义配置"
3. 开启"启用父子分块"开关
4. 设置父分块大小（如 2048）和子分块大小（如 256）
5. 点击"确认"按钮

**预期结果**：
- 父子分块配置正确保存
- 分块时使用父子分块逻辑

## 四、API 测试

### 使用 curl 测试

```bash
# 上传文件并带自定义分块配置
curl -X POST http://localhost:8080/api/v1/knowledge-base/1/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@test.pdf" \
  -F "chunking_config={\"chunk_size\":1000,\"chunk_overlap\":100,\"separators\":[\"\\n\\n\",\"\\n\"],\"enable_parent_child\":false,\"parent_chunk_size\":4096,\"child_chunk_size\":384}"
```

### 验证数据库

```sql
-- 查看文档的分块配置
SELECT id, file_name, chunking_config
FROM knowledges
WHERE knowledge_base_id = 1
ORDER BY created_at DESC
LIMIT 5;
```

## 五、常见问题排查

### 问题 1：对话框按钮无响应

**检查**：
- 浏览器控制台是否有 JavaScript 错误
- `UploadConfigDialog` 组件的 `handleConfirm` 和 `handleCancel` 方法是否正确绑定

### 问题 2：配置未生效（保存为 0 或 NULL）

**检查**：
1. 前端发送的请求中 `chunking_config` 是否为有效的 JSON 字符串
2. 后端是否正确解析 JSON
3. 数据库迁移是否执行成功

**调试方法**：
```bash
# 查看后端日志
tail -f logs/server.log | grep chunking_config
```

### 问题 3：知识库默认配置未预填充

**检查**：
- `kbChunkingConfig` prop 是否正确传递
- 知识库是否有默认配置

## 六、日志验证

查找包含 `chunking_config` 的日志：

```bash
grep "chunking_config" logs/server.log
```

**预期日志**：
```
[INFO] Received chunking_config: {"chunk_size":1000,"chunk_overlap":100,...}
[INFO] Using document-level chunking config
[INFO] Using knowledge base default chunking config
```

### 前端网络请求

打开浏览器开发者工具 → Network 标签：

1. 找到上传文件的请求
2. 检查 FormData 中是否包含 `chunking_config`
3. 值应为 JSON 字符串格式
